# Course Description: BADM 554 - Database Management

## Course Identifier

**Course Code**: BADM 554
**Institution**: University of Illinois
**Instructor**: Vishal Sachdev
**Target Concepts**: 231

## Course Overview

Databases play a critical role in today's business environment. Almost all modern organizations use database technologies to store and manage data in every functional area including operations, finance, accounting, and marketing. This course will equip you with one of the most marketable skills to help businesses better manage their data and discover new opportunities.

## Target Audience

This course is designed for:

- Graduate business students seeking practical database skills
- Business analysts and managers who need to work with data
- Students with basic computer literacy but no prior database experience
- Professionals transitioning into data-driven business roles

## Prerequisites

- Basic computer skills (file management, web browsing)
- Familiarity with spreadsheet applications (Excel, Google Sheets)
- Basic understanding of business concepts
- No prior programming experience required

## Main Topics Covered

### 1. Database Fundamentals (Weeks 1-2)
- Data vs. Information
- Database concepts and terminology
- Database Management Systems (DBMS)
- Database users and roles
- Database system components
- Data models overview

### 2. Relational Model & Tables (Weeks 2-3)
- Relational model concepts
- Tables, rows, and columns
- Keys (primary, foreign, composite)
- Relationships (one-to-one, one-to-many, many-to-many)
- Relational algebra operations
- Set operations

### 3. SQL Fundamentals (Weeks 3-5)
- SQL history and standards
- Data Definition Language (DDL)
- Data Manipulation Language (DML)
- SELECT statements
- WHERE clauses and filtering
- Sorting and ordering
- Aggregate functions (COUNT, SUM, AVG, MIN, MAX)
- GROUP BY and HAVING
- Basic joins (INNER, LEFT, RIGHT)

### 4. Advanced SQL (Weeks 5-7)
- Complex joins and self-joins
- Subqueries (scalar, table, correlated)
- Common Table Expressions (CTEs)
- Window functions
- CASE statements
- String functions and pattern matching
- Date/time functions
- NULL handling

### 5. JSON Data & APIs (Week 7-8)
- JSON structure and syntax
- JSON data types
- Querying JSON in databases
- JSON functions in SQL
- RESTful APIs basics
- API authentication
- Data extraction from APIs
- Working with nested JSON

### 6. Database Design & Modeling (Weeks 8-10)
- Entity-Relationship (ER) modeling
- ER diagrams and notation
- Entities, attributes, and relationships
- Cardinality and participation
- Weak entities
- Enhanced ER (EER) features
- Translating ER to relational schema
- MySQL Workbench for modeling

### 7. Normalization (Week 10-11)
- Data redundancy problems
- Update, insertion, and deletion anomalies
- Functional dependencies
- First Normal Form (1NF)
- Second Normal Form (2NF)
- Third Normal Form (3NF)
- Boyce-Codd Normal Form (BCNF)
- Denormalization trade-offs

### 8. Data Warehousing & ETL (Weeks 11-12)
- OLTP vs. OLAP
- Data warehouse architecture
- Star schema
- Snowflake schema
- Fact tables and dimension tables
- ETL process overview
- ETL tools (Knime)
- Data quality and cleansing
- Data transformation techniques

### 9. NoSQL Databases (Weeks 13-14)
- NoSQL overview and motivation
- CAP theorem
- Document databases (MongoDB)
- Key-value stores
- Column-family stores
- Graph databases
- MongoDB operations (CRUD)
- MongoDB query language
- When to use NoSQL vs. SQL

### 10. Development Environment & Tools
- MySQL installation and configuration
- PostgreSQL setup
- Jupyter Notebooks with Python
- Google Colab environment
- MySQL Workbench
- Knime workflow tool
- MongoDB Compass
- SQL clients and IDEs

### 11. Python for Database Operations
- Database connectivity (libraries)
- Executing SQL from Python
- Parameterized queries
- Transaction management
- Error handling
- Data visualization with database data
- Pandas integration with databases

### 12. Business Applications
- Marketing analytics (customer segmentation)
- Supply chain management (vendor analysis)
- Financial analysis (macroeconomic research)
- Accounting applications (fraud detection)
- Business intelligence dashboards
- Real-world case studies

## Topics Explicitly Excluded

- Advanced database administration (backup/recovery, replication)
- Database security and encryption
- Query optimization and indexing strategies
- Distributed database systems
- Real-time streaming databases
- Blockchain and distributed ledger technologies
- Advanced machine learning integration
- Database performance tuning
- Cloud database migration strategies
- Database as a Service (DBaaS) platforms in depth

## Learning Outcomes

By the end of this course, students will be able to:

1. **Understand and articulate** the data needs of a company (requirement analysis)
2. **Conceptually model** the relationship of data using ER diagrams (data modeling)
3. **Query databases** using SQL to meet business requirements (structured and non-relational)
4. **Design normalized databases** that avoid data anomalies
5. **Extract, Transform, and Load (ETL)** data from/to multiple sources
6. **Work with JSON data** and integrate API data into databases
7. **Understand infrastructure needs** to handle Big Data and build BI workflows
8. **Choose appropriate database technologies** for different business scenarios
9. **Use Python** to interact with databases programmatically
10. **Apply database skills** to solve real-world business problems in marketing, finance, operations, and accounting

## Bloom's Taxonomy Alignment

- **Remember (15%)**: Database terminology, SQL syntax, normalization rules
- **Understand (25%)**: Database concepts, relational model, ER diagrams, data warehousing principles
- **Apply (35%)**: Writing SQL queries, creating ER diagrams, normalizing databases, ETL workflows
- **Analyze (15%)**: Evaluating database designs, identifying data anomalies, choosing appropriate technologies
- **Evaluate (5%)**: Assessing database performance, comparing SQL vs NoSQL
- **Create (5%)**: Designing complete database solutions for business problems

## Assessment Methods

- Weekly SQL assignments
- Database design projects
- ETL workflow implementation
- Final capstone project integrating multiple concepts
- Quizzes on foundational concepts

## Course Philosophy

This course emphasizes **practical, hands-on learning** with real business scenarios. Every concept is reinforced with:

- Business context and motivation
- Interactive examples
- Hands-on labs
- Real-world datasets
- Industry-relevant tools

The goal is for students to leave the course with immediately applicable skills that can be used in their careers.
